## 1. 프로젝트 소개
![소개이미지](https://github.com/user-attachments/assets/a4ad43bc-7ee0-4608-b150-43b712888b68)

🔃손쉬운 조작

자동 전투를 통해 방치만 해도 재화를 획득해 영웅을 성장시켜 모험을 진행할 수 있습니다.

📜다양한 동료

매력적인 스킬을 사용하는 다양한 동료를 모아 팀을 구성할 수 있습니다. 나만의 팀을 구성하여 키워보세요! ⏩초고속 성장

🎯 강력한 스킬

잠겨진 8가지의 멋지고 강력한 스킬을 해금하여 아주 강력한 적도 물리칠 수 있어요! 💪

⚔️장비 뽑기 시스템

귀여운 미믹을 마석을 먹여 강력한 무기를 뽑을 수 있습니다. 높은 등급의 장비 아이템을 뽑아 보다강한 히어로로 만들어 보아요!

👾던전 시스템

몰려오는 몬스터들을 제압하여 던전을 공략하면 보다 많은 재화를 얻을 수 있어요!

장비 강화, 스텟 강화, 스킬 획득 등 다양한 방법을 통해 강해질 수 있습니다. 다양한 콘텐츠와 지속적인 보상 시스템을 통해 초고속 성장을 경험할 수 있습니다.

## 6. 트러블 슈팅

<details>
 <summary>Fade 효과 중에 내부에서 코루틴 실행 서순 문제</summary>
 
 * 문제 : Fade 효과 코루틴을 실행하면서 Fade중에 화면이 다 가려지지 않았는데도 
 변화가 눈에 보이는 문제가 생김
 
 * 원인 : Fade 함수 호출 이후 바로 코루틴을 호출하면서 문제가 발생
  
 * 해결 : 콜백 함수를 사용 하여 코루틴 함수를 콜백으로 받아 Fade 함수에서 코루틴으로 호출
 코루틴 함수가 완료 될 때 까지 yield return으로 대기하여 의도한 화면 전환 효과를 얻음

   [UILoading.cs > StartFadeCoroutine 함수]()
  
</details>

<details>
<summary>동료 모집 진행 중 창을 닫아버리면 정보가 사라지는 현상</summary>

* 문제 : 동료 모집 10회 모집 진행 중 동료 모집 UI창을 닫아버리면 더이상 동료 모집을 진행할 수 없는 문제 발생

* 원인 : 동료를 하나씩 공개하는 코루틴에 가챠 결과를 넣어 코루틴이 실행 되는 중 창을 닫으면 코루틴이 강제 종료되어 해당 문제가 발생하였음.

* 해결 : 동료 모집 버튼을 클릭하였을 때 가챠 결과를 이미 결정하고, UI 상에서 보여주는 것만 코루틴 처리하여 창을 닫더라도 결과가 정상적으로 반영되도록 수정

  [링크]()
 
</details>

<details>
<summary>캐릭터 FSM에서 동시에 상태를 바뀌지 않는 문제</summary>

* 문제 : 캐릭터들의 스테이트 변화를 일괄적으로 변환하려 했으나 캐릭터 스테이트 내에서는 일괄적으로 다루기 힘든 부분이 존재

* 해결 : GamaManager에서 전투 상태를 저장하고 캐릭터에서 GamaManager의 전투 상태를 참조하여 상태가 변경되도록 변경
 
</details>

<details>
<summary>이미지 렌더링이 y축 기준으로 정렬 안되는 문제</summary>

* 문제 : 2D 벨트 스크롤 형식의 맵 스크롤링과 캐릭터 움직임을 구현하려 했으나 이미지 렌더링이 Z축 기준으로 정렬이 되어서 화면 상에서 이미지가 뒤죽박죽으로 렌더링 되는 문제가 생김

* 원인 : Transparency sort mode에서 정렬 기준이 Z축으로 되어서 Order in Layer 설정해주지 않으면 Z축 값을 기준으로 렌더링이 됨

* 해결 : Project Settings > Graphics > Transparency sort mode 에서 Custom Mode로 변경 후 Z 값을 0 Y 값을 1로 두어 Y축 값을 기준으로 이미지가 렌더링 되도록 함

  ![image](https://github.com/user-attachments/assets/45c0f95f-2651-4d0f-b657-42705eb9714a)
 
</details>

<details>
<summary>UI : 모바일 해상도 대응</summary>

* 문제 : 다양한 해상도에서 UI 요소가 예상과 다르게 배치되거나 크기가 조정됩니다.

* 해결 : Canvas Scaler의 설정을 Expand 모드로 변경하여, 모든 해상도에서 UI 요소가 화면을 최대한 채우도록 합니다.

  ![image](https://github.com/user-attachments/assets/84f63369-4837-48cb-8002-73aee9d573d6)
 
</details>

<details>
<summary>UI : 애니메이션 Trigger가 두 번 실행되는 문제</summary>

* 문제 : Mimic 애니메이션 Trigger가 의도치 않게 두 번 실행되는 문제가 있었습니다. 이 문제는 Trigger를 설정하는 방식과 Trigger가 호출되는 타이밍이 적절하지 않아서 발생했습니다.<br/>
구체적으로는 중복 호출이 있었는데 애니메이션 Trigger를 설정하는 메서드나 로직이 여러 번 호출될 수 있는 구조였기 때문에, 단일 이벤트에서 애니메이션이 두 번 실행되는 현상이 발생했습니다.

* 해결 : MimicAnimationController 스크립트를 사용하여 애니메이션 트리거 설정을 직접 관리하도록 변경했습니다. 이 스크립트는 애니메이션이 트리거되기 전에 특정 조건을 확인하여, 필요할 때만 트리거를 설정할 수 있도록 했습니다.

  [MimicAnimationController.cs > SetTrigger(int index)]()
 
</details>
